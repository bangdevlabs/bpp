# font_forge — B++ font pipeline

The B++ canonical font lives as a human-editable text file
(`assets/fonts/b++_font.txt`) and converts through this folder into
a ModuLab project (for visual refinement) and eventually into a raw
bitmap suitable for baking directly into `stb/stbdraw.bsm` next to
the default 8x8 fallback.

**Cobra comendo o próprio rabo**: ModuLab paints pixel art;
B++ fonts are pixel art; so the font of the B++ ecosystem is
designed in ModuLab, edited in ModuLab, exported by a B++ tool,
and baked into a B++ stdlib module. The whole chain is in-house.

## Lineage

| Source                    | Trait inherited                        |
|---------------------------|----------------------------------------|
| Bell Labs Plan 9 terminals | Monospace 8-wide, programmer-facing   |
| IBM VGA 8x16 text mode    | Row stride, overall proportions       |
| LucasArts SCUMM dialogue  | Slight warmth on curves (pixel-step)  |
| Sierra Space Quest HUD    | Angular, pointed diagonals            |
| Modern programming fonts  | Slashed zero, spurred G, distinct i/l/1 |

The synthesis is a terminal-era bitmap font with tiny humanist
touches — readable at 16px, crisp at integer multiples (32, 48),
no antialiasing ever.

## Layout

```

tools/font_forge/

  export_fnt.bpp            (TODO) .modulab.json → .fnt bitmap
  README.md                 this file
```

## Glyph grid

- 8 columns × 16 rows per glyph.
- ASCII 32-127 (96 codepoints).
- Coordinate convention:
    - row 0–1: top margin
    - row 2: top of caps / ascenders (e.g. top of `A`, `B`, `T`, `l`)
    - row 6: top of x-height (top of `a`, `e`, `o`, `n`)
    - row 11: baseline (bottom of `A`, `a`, `O`)
    - row 12–14: descender space (`g`, `j`, `p`, `q`, `y`)
    - row 15: bottom margin

In the ModuLab project, the canvas is 16×16 (ModuLab currently
requires square canvases). The glyph lives in cols 0–7; cols 8–15
stay blank as breathing room.

## Workflow

### Round trip: hand-edit → refine visually

```
# 1. Tweak the text source (fast path)
$EDITOR assets/fonts/b++_font.txt

# 2. Convert to ModuLab project
./bpp tools/font_forge/txt_to_modulab.bpp         \
      assets/fonts/b++_font.txt                   \
      assets/fonts/b++_font.modulab.json

# 3. Open in ModuLab for visual tweaks
./tools/modulab/build/modulab assets/fonts/b++_font
# ModuLab loads b++_font.modulab.json automatically.
# Frame N corresponds to ASCII codepoint 32+N.
# Frame 0 = space, frame 33 = A, frame 65 = a, etc.

# 4. (Future) Bake into the runtime
./bpp tools/font_forge/export_fnt.bpp             \
      assets/fonts/b++_font.modulab.json          \
      stb/stbdraw_bpp_font.data
# Then either commit the byte array inline into stbdraw.bsm
# or load it on startup from disk.
```

### Starting from scratch

If you delete `b++_font.txt`, you lose the text seed. Use
ModuLab's export to regenerate it — a future `modulab_to_txt.bpp`
will dump any 16×16, 96-frame project back to the `.txt` format.
(The round trip is lossless: only palette indices 0 and 1 carry
through; all other indices collapse to 1 on the bitmap side.)

## Design rules of thumb

- **Stroke width = 1 pixel**. Everywhere. Double-strokes read muddy
  at 8px cell size.
- **Symmetry on caps**: `A`, `M`, `O`, `V`, `W`, `X` are left-right
  mirror symmetric. Check each when editing.
- **Diagonals step 1:1**: a 45° stroke moves 1 pixel down per 1
  pixel across. `N`, `Z`, `/` all share this rhythm.
- **Baseline discipline**: every cap bottom sits on row 11. Lowercase
  x-height tops sit on row 6. Descenders end at row 14 latest.
- **Readability over personality**: if a glyph is cute but hard to
  read at 16px, make it boring. The font is infrastructure, not
  branding. (The ecosystem *context* — ModuLab, Wolf3D HUD —
  brings the personality.)

## Programmer-friendly features

- `0` has a diagonal slash through it so it's never mistaken for `O`.
- `G` has a horizontal spur on the inside so it reads distinctly
  from `C` at small sizes.
- `1` has a top-left serif and a bottom crossbar so it won't be
  read as `l` or `I`.
- `I` has top and bottom serifs so it stands apart from `l` (no
  serifs) and `1`.
- `l` is a plain vertical with a tiny foot to the right — just
  enough difference from `I` to be unambiguous.
- `i` has a dot and a tiny foot to mirror `l`.

## Future work

- **export_fnt.bpp**: convert the `.modulab.json` back into a
  flat 1.5 KB bitmap, baked as a `const` byte array inside
  `stb/stbdraw.bsm` as the optional `b++_8x16` font. Users can
  still pick the 8×8 default, or this, via `font_load`.
- **Non-ASCII coverage**: add common accented Latin (á é í ó ú ñ ç)
  when a game or tool needs it. The slot range 128-255 is
  available in the parser if we extend `ASCII_LAST`.
- **BDF / TTF export**: if a third-party tool ever needs the font,
  a BDF writer is ~150 LOC. Run the BDF through `fontforge` once,
  commit the resulting TTF — no in-tree TTF emitter required.
